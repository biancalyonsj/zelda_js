<h1 align="center">Zelda Browser Game</h1>

![image alt](https://github.com/biancalyonsj/zelda_js/blob/e2d031c7bff79fab50890b8b35dd325166aafff5/zelda_homescreen.jpg)

<p>Created a simple recreation of the classic Zelda game using JavaScript</p>

<h2>Concepts Learned</h2>
<h3>DOM Manipulation & Dynamic UI Rendering</h3>
<li>Built a fully interactive grid-based game interface using the DOM API</li>
<li>Dynamically generated the game board by mapping structured data into DOM elements</li>
<li>Programmatically created, updated, and removed elements to reflect real-time game state</li>
<li>Managed UI updates for score, level progression, and enemy count using synchronized DOM updates</li>

<h3>Data-Driven Architecture & Grid Systems</h3>
<li>Designed a grid-based rendering system using a 2D map representation</li>
<li>Translated character-based map data into visual UI components (walls, doors, enemies, objects)</li>
<li>Implemented coordinate-to-index mapping for efficient grid calculations and movement logic</li>
<li>Separated map data from rendering logic for scalability and maintainability</li>

<h3>Game Loop & Time-Based Animation</h3>
<li>Implemented a performant game loop using requestAnimationFrame</li>
<li>Utilized delta time (deltaTime) calculations to ensure consistent animation across varying frame rates</li>
<li>Controlled enemy movement and game updates independently of frame speed</li>

<h3>Event Handling & User Interaction</h3>
<li>Built a responsive input system using keyboard event listeners</li>
<li>Handled multi-directional movement and action inputs (attack mechanics)</li>
<li>Coordinated asynchronous behavior with game state for smooth user interactions</li>

<h3>Asynchronous JavaScript & Timers</h3>
<li>Used setTimeout for timed UI effects and temporary elements</li>
<li>Managed animation lifecycles to ensure proper cleanup of DOM nodes</li>
<li>Coordinated asynchronous behavior with game state updates</li>

<h3>Reusability & Code Organization</h3>
<li>Structured code into modular functions for Board Creation, Movement logic, Enemy behavior, UI updates</li>
<li>Emphasized reusable logic for scalability and easier feature expansion</li>
